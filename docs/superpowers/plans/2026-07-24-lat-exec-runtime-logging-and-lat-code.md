# LAT Exec Runtime Logging and lat-code Rename Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Persist timestamped, channel-separated stdout/stderr runtime logs for external Codex and Claude exec clients so a later Dispatch session can diagnose transient errors, and rename the `lat-runner` skill to `lat-code` with the shared YAML schema moved into `lat-dispatch/references/`.

**Architecture:** `scripts/run-exec-client.sh` (the "exec launcher") grows an explicit interface (`--stdout-log`, `--stderr-log`, `--agent-id`, `--action launch|resume`, `--stdout-format`) and pipes the client's FD1/FD2 through per-line UTC timestamp capture into `.lat/logs/<TASK_ID>/<agent_id>.stdout.jsonl` and `<agent_id>.stderr.log`, with boundary records per launch/resume. Session JSONL via `monitor-session.sh` remains the sole completion/Final-Answer source; runtime logs are diagnosis-only with a progressive `tail -n 7` → `tail -n 20` ladder. The `lat-runner` skill becomes `lat-code`; its shared `yaml-schema.md` moves to `lat-dispatch/references/` because it defines the ledger for code, test, and QA executors.

**Tech Stack:** Bash 3.2+ (GNU/Linux and macOS), jq, trash-cli, existing LAT shell test suites, Markdown skills per https://agentskills.io/skill-creation/best-practices.

## Global Constraints

- All file deletion uses `trash-put` (trash-cli); never `rm` for user-visible files.
- Bash must run on GNU/Linux and macOS with Bash 3.2+: no `wait -n`, no `mapfile`, no GNU-only `date`/`stat`/`awk` features in new code (`date -u +%Y-%m-%dT%H:%M:%SZ` is portable).
- Use GitHub CLI (`gh`) for any GitHub operation; use `uv` if Python is ever needed (none is planned).
- Session JSONL remains the only source for turn completion and Final Answer; runtime stdout must never be used to decide completion.
- FD1 and FD2 are captured to separate files; never merge with `2>&1`; no cross-client common event schema — native events go untouched inside an `event` field.
- `captured_at` is Dispatch-host RFC 3339 UTC seconds (e.g. `2026-07-24T09:42:18Z`), generated by the LAT capture layer.
- PID file lifecycle unchanged: `.lat/workspace/<TASK_ID>/runtime/<agent_id>.pid`, exact client PID only, trashed by the exec launcher on exit. Runtime logs live in `.lat/logs/<TASK_ID>/` and survive client exit under `logs.retention_days` (default 60).
- New Agent Sessions use a new `agent_id` and must never overwrite existing runtime logs; resume appends to the original files with a timestamped boundary in both channels.
- Never escalate to `SIGKILL`; never locate processes with `pgrep`; on confirmed capture-channel failure send exactly one `SIGTERM` to the saved client PID.
- Active documents must not call the exec launcher a bare "runner" (reserved for avoiding confusion with the Agent skill). Per the client-naming convention, exec client references say "exec", not "CLI client" where the exec form is meant.
- Historical specs/plans under `docs/superpowers/` and `todo/` are never edited; unrelated dirty files are preserved.
- `agent_id` format `code_executor_<instance>_<task_id>`, phase names, ledger fields, status transitions, and results-first reconciliation are unchanged.
- Skill files must follow agentskills.io best practices: frontmatter `name` matches the directory, `SKILL.md` under 500 lines, progressive disclosure via references, and no links to repo-root `docs/`. The one sanctioned cross-skill reference is `lat-code` → `../lat-dispatch/references/yaml-schema.md` (works in both the repo and the installed `~/.agents/skills/` layout), declared as a companion dependency.
- Skill/reference documents stay in Traditional Chinese (existing language); tests, commit messages, and this plan are English.
- Resume keeps the original model, effort, permission, and the existing Session JSONL `--after-line` contract.

Exit codes for the exec launcher (documented in `clients.md`): `64` usage, `65` refused start (existing PID file, existing runtime log on `launch`, or missing runtime log on `resume`), `69` missing dependency, `70` capture-channel failure (detected while the client runs **or** from a nonzero capture exit status after client EOF), `73` filesystem failure.

Launcher fatal diagnostics have a single canonical source at every point in time: before the runtime logs exist (usage, missing dependency, directory failure, refused launch/resume, boundary write), errors go to the launcher's own stderr because there is nowhere else to put them; once `RUNTIME_LOG_READY=true`, launcher/capture fatal diagnostics are written **only** as timestamped `<UTC> LAT_LAUNCHER_ERROR agent_id=<agent_id> <message>` lines in the stderr runtime log and are never duplicated to the shell — the Dispatch shell is reserved for Monitor lifecycle output. Backgrounded examples keep launcher FD2 attached solely so pre-log failures stay visible.

Replace the file body with (keep the shebang and `set -uo pipefail`):

```bash
#!/usr/bin/env bash

set -uo pipefail

usage() {
  echo "Usage: $0 --pid-file PATH --stdout-log PATH --stderr-log PATH --agent-id ID --action launch|resume --stdout-format codex-jsonl|claude-stream-json -- command [args...]" >&2
  exit 64
}

PID_FILE=""
STDOUT_LOG=""
STDERR_LOG=""
AGENT_ID=""
ACTION=""
STDOUT_FORMAT=""

while [ "$#" -gt 0 ]; do
  case "$1" in
    --) shift; break ;;
    --pid-file|--stdout-log|--stderr-log|--agent-id|--action|--stdout-format)
      [ "$#" -ge 2 ] || usage
      case "$1" in
        --pid-file) PID_FILE=$2 ;;
        --stdout-log) STDOUT_LOG=$2 ;;
        --stderr-log) STDERR_LOG=$2 ;;
        --agent-id) AGENT_ID=$2 ;;
        --action) ACTION=$2 ;;
        --stdout-format) STDOUT_FORMAT=$2 ;;
      esac
      shift 2
      ;;
    *) usage ;;
  esac
done

[ "$#" -gt 0 ] || usage
[ -n "$PID_FILE" ] || usage
[ -n "$STDOUT_LOG" ] || usage
[ -n "$STDERR_LOG" ] || usage
[[ "$AGENT_ID" =~ ^[A-Za-z0-9._-]+$ ]] || usage
case "$ACTION" in launch|resume) ;; *) usage ;; esac
case "$STDOUT_FORMAT" in codex-jsonl|claude-stream-json) ;; *) usage ;; esac
command -v trash-put >/dev/null || { echo "ERROR: trash-put not found" >&2; exit 69; }
command -v jq >/dev/null || { echo "ERROR: jq not found" >&2; exit 69; }

mkdir -p "$(dirname "$PID_FILE")" || exit 73
mkdir -p "$(dirname "$STDOUT_LOG")" "$(dirname "$STDERR_LOG")" || exit 73
[ ! -e "$PID_FILE" ] || { echo "ERROR: PID file already exists: $PID_FILE" >&2; exit 65; }
if [ "$ACTION" = launch ]; then
  for log in "$STDOUT_LOG" "$STDERR_LOG"; do
    [ ! -e "$log" ] || { echo "ERROR: runtime log already exists for a new launch: $log" >&2; exit 65; }
  done
else
  # resume appends to the original files; it must never create them.
  for log in "$STDOUT_LOG" "$STDERR_LOG"; do
    [ -f "$log" ] || { echo "ERROR: resume requires an existing runtime log: $log" >&2; exit 65; }
  done
fi

utc_now() { date -u +%Y-%m-%dT%H:%M:%SZ; }

RUNTIME_LOG_READY=false
launcher_diag() {
  # Single canonical diagnostic source: before the runtime logs exist, fatal
  # errors go to the launcher's own stderr; once the stderr runtime log is
  # ready, fatal diagnostics are written only there (never duplicated to the
  # shell — the Dispatch shell is reserved for Monitor lifecycle output).
  local msg=$1
  if [ "$RUNTIME_LOG_READY" = true ]; then
    printf '%s LAT_LAUNCHER_ERROR agent_id=%s %s\n' "$(utc_now)" "$AGENT_ID" "$msg" \
      >>"$STDERR_LOG" 2>/dev/null || echo "ERROR: $msg" >&2
  else
    echo "ERROR: $msg" >&2
  fi
}

FIFO_DIR=$(mktemp -d "${TMPDIR:-/tmp}/lat-exec-capture.XXXXXX") || exit 73
CLIENT_PID=""
OUT_CAP_PID=""
ERR_CAP_PID=""

# shellcheck disable=SC2317 # Invoked indirectly by the EXIT trap.
cleanup() {
  [ ! -e "$PID_FILE" ] || trash-put -- "$PID_FILE"
  [ ! -d "$FIFO_DIR" ] || trash-put -- "$FIFO_DIR"
}
trap cleanup EXIT

mkfifo "$FIFO_DIR/stdout" "$FIFO_DIR/stderr" || exit 73

BOUNDARY_TS=$(utc_now)
jq -cn --arg ts "$BOUNDARY_TS" --arg agent_id "$AGENT_ID" --arg action "$ACTION" \
  '{captured_at: $ts, stream: "meta",
    event: {type: "lat.runtime_boundary", agent_id: $agent_id, action: $action}}' \
  >>"$STDOUT_LOG" || { echo "ERROR: cannot write stdout runtime log" >&2; exit 73; }
printf '%s LAT_RUNTIME_BOUNDARY agent_id=%s action=%s\n' \
  "$BOUNDARY_TS" "$AGENT_ID" "$ACTION" \
  >>"$STDERR_LOG" || { echo "ERROR: cannot write stderr runtime log" >&2; exit 73; }
RUNTIME_LOG_READY=true

# stdout capture: one envelope per line; non-JSON lines are kept as unparsed events.
jq -Rc --unbuffered '. as $line
  | {captured_at: (now | todate), stream: "stdout",
     event: ($line | try fromjson catch {type: "lat.unparsed_line", text: $line})}' \
  <"$FIFO_DIR/stdout" >>"$STDOUT_LOG" &
OUT_CAP_PID=$!

capture_stderr() {
  local line
  while IFS= read -r line || [ -n "$line" ]; do
    printf '%s %s\n' "$(date -u +%Y-%m-%dT%H:%M:%SZ)" "$line"
  done
}
capture_stderr <"$FIFO_DIR/stderr" >>"$STDERR_LOG" &
ERR_CAP_PID=$!

if ! kill -0 "$OUT_CAP_PID" 2>/dev/null || ! kill -0 "$ERR_CAP_PID" 2>/dev/null; then
  launcher_diag "timestamp capture failed before client start"
  kill -TERM "$OUT_CAP_PID" "$ERR_CAP_PID" 2>/dev/null || true
  wait "$OUT_CAP_PID" "$ERR_CAP_PID" 2>/dev/null || true
  exit 70
fi

"$@" <&0 >"$FIFO_DIR/stdout" 2>"$FIFO_DIR/stderr" &
CLIENT_PID=$!
if ! printf '%s\n' "$CLIENT_PID" >"$PID_FILE"; then
  launcher_diag "cannot write PID file: $PID_FILE"
  kill -TERM "$CLIENT_PID" 2>/dev/null || true
  wait "$CLIENT_PID" 2>/dev/null || true
  wait "$OUT_CAP_PID" "$ERR_CAP_PID" 2>/dev/null || true
  exit 73
fi

CAPTURE_FAILED=false
while kill -0 "$CLIENT_PID" 2>/dev/null; do
  if ! kill -0 "$OUT_CAP_PID" 2>/dev/null || ! kill -0 "$ERR_CAP_PID" 2>/dev/null; then
    CAPTURE_FAILED=true
    # Single SIGTERM, only because the exact client PID was just seen alive.
    kill -TERM "$CLIENT_PID" 2>/dev/null || true
    break
  fi
  sleep 0.2
done

wait "$CLIENT_PID"
STATUS=$?
wait "$OUT_CAP_PID"
OUT_CAP_STATUS=$?
wait "$ERR_CAP_PID"
ERR_CAP_STATUS=$?

# A capture can also fail concurrently with client exit; validate both capture
# statuses after draining instead of trusting only the liveness loop.
if [ "$CAPTURE_FAILED" = false ] && { [ "$OUT_CAP_STATUS" -ne 0 ] || [ "$ERR_CAP_STATUS" -ne 0 ]; }; then
  CAPTURE_FAILED=true
fi
if [ "$CAPTURE_FAILED" = true ]; then
  launcher_diag "capture channel failed (stdout=$OUT_CAP_STATUS stderr=$ERR_CAP_STATUS client=$STATUS)"
  exit 70
fi
exit "$STATUS"
```

Design notes for the implementer: the stdout capture runs `jq` as a direct child of the launcher (not wrapped in a function) so the mid-run failure test in Task 3 can target it; captures start and are verified alive before the client starts; the client exiting closes both FIFO write ends, so both captures see EOF and drain the tail before the final `wait` — that is the tail-integrity guarantee. Both capture exit statuses are collected after draining and any nonzero result forces exit 70, so a capture failing concurrently with client exit cannot silently return the client status; the single `SIGTERM` is sent only from the liveness loop, immediately after the exact client PID was seen alive. `launcher_diag` enforces a single canonical diagnostic source: pre-log failures go to the launcher's own stderr (no log exists yet), and after the `RUNTIME_LOG_READY` gate every fatal launcher/capture diagnostic is written only to the stderr runtime log, never duplicated to the shell (the launcher-stderr fallback fires only if the log itself became unwritable). `--action resume` refuses to start (exit 65) unless both runtime logs already exist as regular files — resume appends to the originals and never creates them. `--stdout-format` is validated but both accepted values use the same JSONL-per-line capture; the flag exists so callers state the native format explicitly instead of the launcher guessing from file names.

- [ ] **Step 1: Staged, rollback-safe replacement — old trees survive until the new ones are verified in place**

This is a staged swap, not an atomic one: both the backup loop and the two final `mv` operations can be interrupted halfway, so rollback must know **per skill** whether the original final path was present (and moved to backup), absent, or not yet processed at all. That state is recorded inside `$STAGE/backup` as the processing marker: for a present original, the moved backup tree itself is the marker (the `mv` that creates it is the same operation that vacates the final path, so an interruption can never misclassify an untouched original); for an absent original, a `<skill>.absent` marker file is written. A skill with neither marker was never processed and its final path is left untouched. Only after the installed trees verify byte-for-byte are the backups and stage trashed. Bash 3.2-compatible; no `rm`, no delete flags; unrelated installed skills are never touched.

```bash
ROOT=/home/swy/.agents/skills
STAGE=$(mktemp -d "$ROOT/.lat-skill-stage.XXXXXX") || exit 1
mkdir -p "$STAGE/new" "$STAGE/backup" || exit 1
SWAP_DONE=false

# Per-skill processing markers in $STAGE/backup:
#   backup/<skill>         (directory) original was present and moved here
#   backup/<skill>.absent  (file)      original final path did not exist
# No marker means the skill was never processed and its final path is the
# untouched original.

# shellcheck disable=SC2317 # Invoked indirectly by the EXIT trap.
rollback() {
  [ "$SWAP_DONE" = true ] && return 0
  local skill
  for skill in lat-dispatch lat-code lat-runner; do
    if [ -e "$STAGE/backup/$skill" ]; then
      # Processed, original present: anything now at the final path is a
      # (possibly partial) new tree — trash it, then restore the backup.
      [ ! -e "$ROOT/$skill" ] || trash-put "$ROOT/$skill"
      mv "$STAGE/backup/$skill" "$ROOT/$skill"
    elif [ -e "$STAGE/backup/$skill.absent" ]; then
      # Processed, original absent: trash any partially installed new tree
      # and leave the final path absent, matching the pre-swap state.
      [ ! -e "$ROOT/$skill" ] || trash-put "$ROOT/$skill"
    fi
    # No marker: unprocessed — the final path still holds the untouched
    # original (or was already absent); leave it alone.
  done
  echo "ROLLED BACK: pre-swap state restored; inspect $STAGE, then trash-put it" >&2
}
trap rollback EXIT

# 1. Stage the repo trees and verify them byte-for-byte before touching
#    anything installed.
cp -R lat-dispatch "$STAGE/new/lat-dispatch" || exit 1
cp -R lat-code "$STAGE/new/lat-code" || exit 1
diff -qr lat-dispatch "$STAGE/new/lat-dispatch" || exit 1
diff -qr lat-code "$STAGE/new/lat-code" || exit 1

# 2. Backup phase with markers. For a present original the mv itself records
#    the state; the .absent marker is written only when there is nothing to
#    move, so no interruption point can misclassify an untouched original.
for skill in lat-dispatch lat-code lat-runner; do
  if [ -e "$ROOT/$skill" ]; then
    mv "$ROOT/$skill" "$STAGE/backup/$skill" || exit 1
  else
    : >"$STAGE/backup/$skill.absent" || exit 1
  fi
done

# 3. Move the verified new trees into place, dependency-safe order:
#    lat-dispatch first, because lat-code references its shared schema.
mv "$STAGE/new/lat-dispatch" "$ROOT/lat-dispatch" || exit 1
mv "$STAGE/new/lat-code" "$ROOT/lat-code" || exit 1

# 4. Final verification of the installed trees before anything is discarded.
diff -qr lat-dispatch "$ROOT/lat-dispatch" || exit 1
diff -qr lat-code "$ROOT/lat-code" || exit 1

# 5. Success: only now retire the backed-up old trees, markers, and stage.
SWAP_DONE=true
trash-put "$STAGE"
echo "SYNC OK"
```

Rollback restores the exact pre-swap state in every failure window: a partial **backup** failure restores the skills already moved and leaves unprocessed ones untouched; a partial **install** failure trashes whichever new trees reached their final paths (including a new `$ROOT/lat-code` on a host that never had an old `lat-code` — its `.absent` marker makes rollback remove it rather than keep it); a **final diff** failure trashes both installed new trees and restores or leaves-absent each skill per its marker. After a rolled-back run, inspect the reported `"$ROOT"/.lat-skill-stage.*` directory and clean it with `trash-put` (never `rm`).
